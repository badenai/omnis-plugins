## Core Knowledge (by weight)

Below is the codified doctrine for translating market structure into deterministic machine rules.

### I. Execution Architecture & Finite State Machines
Human execution is flawed by latency and emotion. Trade lifecycle management must exist exclusively within event-driven architectures like [NautilusTrader](concepts/event-driven-execution-engines.md), optimized for high-performance tick data consumption and state transition logic.
*   **FSM State Closure:** Subjective trade management is invalid. A trade is a mathematically "closed" Finite State Machine where entries, partial fills, trailing stops, and exits are explicit deterministic states. The system must employ error-aware rollbacks to map every potential asynchronous API error (e.g., partial fills, disconnects) to a defined reactive state, eliminating undefined deadlock behaviors [Trade Lifecycle Validation](concepts/trade-lifecycle-validation.md).
*   **Reinforcement Learning MDPs:** Market environments must be wrapped into Markov Decision Processes (MDP). Proximal Policy Optimization (PPO) agents map derived technical state matrices to discrete, quantifiable actions (Buy/Sell/Hold) relying strictly on a balance-delta reward function [RL Environment Architecture](concepts/rl-environment-architecture.md).

### II. Structural Mechanics & Price Action Formalization
Chart patterns must be converted into strict geometric or regression-based models. "Trend" is a useless term unless defined computationally.
*   **FVG RANSAC Regression:** Fair Value Gaps (FVGs) are graded using RANSAC regression on 1-second tick data during macro formation. The generation velocity is the absolute slope ($|\beta1|$). Only low-degree FVGs (flatter slopes, $|\beta1| \approx 0$) pass qualification as genuine structural consensus zones; high-degree steep FVGs are invalidated as exhaustion noise [FVG Tick-Level RANSAC Degree](concepts/fvg-tick-level-ransac-degree.md).
*   **Volatility-Adjusted Gap Memory:** Gap significance is verified by scaling against Average True Range (ATR). FVGs formed in low-volatility are filtered out. "Gap Memory" state dynamically dictates how long a zone is structurally active, creating adaptive trailing support/resistance arrays [Volatility-Scaled FVG Memory](concepts/volatility-scaled-fvg-memory.md).
*   **Recursive Fractal Mapping:** Multi-timeframe structures are modeled as concurrent FSMs. Higher-timeframe (HTF) pivot arrays act as rigid regime filters, while lower-timeframe Change of Character (CHoCH) acts as a local trigger. Entries target the exact 50% midpoint (Consequent Encroachment) of order blocks with deterministic invalidation [Recursive Fractal Structure Mapping](concepts/recursive-fractal-structure-mapping.md).
*   **Liquidity Sweeps & Invalidation:** Sweeps are defined via a lookback window, sensitivity variance threshold, and programmatic confirmation (e.g., Close > Open following the sweep). Walk-Forward Analysis (WFA) dynamically updates the sensitivity threshold [Liquidity Sweep Walk-Forward Validation](concepts/liquidity-sweep-walk-forward-validation.md).
*   **Structural Failures:** Retention failures and "Zones of Failure" are defined by integer-touch sequence counters. The algorithm only triggers on the Nth failure (e.g., the 3rd touch failure to gain), targeting mean reversion [Structural Failure Patterns](concepts/structural-failure-patterns.md). Buyer-Seller Flips require absolute confirmation: a historical level is breached, generating a new extreme pivot, followed by a retest rejection [Market Structure Logic](concepts/market-structure-logic.md).

### III. Quantitative Volume & Microstructure Mechanics
Volume validates structure. Without volumetric confirmation, price expansion is an unverified hypothesis.
*   **Volume Z-Scored Divergence:** Divergence is programmatically defined. If price creates higher highs but High Volume Nodes (HVNs) contract, a Volume Z-score $> 2$ triggers an automatic short signal representing structural exhaustion [Algorithmic Volume Profile Validation](concepts/algorithmic-volume-profile-validation.md).
*   **POC Migration Logic:** Uptrends are only authorized when the Point of Control (POC) migration counter detects $\geq3$ consecutive positive shifts concurrent with Value Area (VA) expansion [Volume Profile Structural Logic](concepts/volume-profile-structural-logic.md).
*   **Microstructure Liquidation Mapping:** Open Interest (OI) Delta mapped against price kinematics yields a continuous divergence score. When matched with extreme funding rate flips, algorithms explicitly compute liquidation cascade "flush" targets as highly-probabilistic limit-order exit nodes [Market Microstructure Liquidity](concepts/market-microstructure-liquidity.md).
*   **Smart Money Proxies:** Institutional presence is computationally modeled using feature engineering: Price-Volume Interaction (`Daily Return * (Current Volume / 50-day SMA Volume)`) and internal bar position (`(Close - Open) / (High - Low)`). RVOL must be $> 1.5x$ on structural breaks to permit execution [Volume Price Interaction Proxies](concepts/volume-price-interaction-proxies.md).

### IV. Risk Engine Encoding
Risk is the only variable the machine truly controls. Every signal generated without programmatic invalidation is a critical architecture failure.
*   **Hardcoded Expectancy:** Fixed risk is bound between $0.1\%$ and $1.0\%$ of portfolio equity. The algorithm targets a minimum 1:4 risk-to-reward ratio (or strict 1:3 minimum), generating positive expectancy even with a baseline 40% win rate [Risk Engine and Liquidity Mapping](concepts/risk-engine-and-liquidity-mapping.md).
*   **Range Midpoint Scaling:** For retracement entries, positions scale exponentially from the 0.5 (midpoint) origin to the 1.0 boundary, drastically improving average entry price and optimizing the R:R geometry before the hard boundary stop [Range Midpoint Scaling Algorithm](concepts/range-midpoint-scaling-algorithm.md).
*   **Correlation Elasticity Doctrine:** The risk engine continuously calculates system-wide Correlation Elasticity. When independent strategies begin moving in lockstep during structural distress, an automated Exposure Compression Protocol triggers, reducing leverage across all FSMs [Risk Management Logic](concepts/risk-management-logic.md).
*   **Vectorized Cross-Asset Validation:** Models must be backtested via vectorized engines (VectorBT) across large cross-asset datasets (e.g., 47 assets, 10 years). Sharpe Ratio and Maximum Drawdown are optimized at the portfolio level [Backtesting Validation Frameworks](concepts/backtesting-validation-frameworks.md).
